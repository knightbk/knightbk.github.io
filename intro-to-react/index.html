The “Old” Way

Using jQuery or vanilla JavaScript we would:

  • Write some HTML.
  • In the JavaScript, write selectors for various buttons, form elements, and DOM elements that exist in the HTML.
  • Listen for events on those elements - click, mouseover, scroll, double-click, keypress, etc.
  • Write some code to react to those events - change the DOM as needed.

Take a look at this example.

Disadvantages of the “Old” Way

  • Gets complex pretty quickly
  • Lots of forks - if/else if/else if/else
  • Easy to introduce bugs

The React Way

  • Made up of individual reusable components.
  • Event handlers update the state.
  • Components render and re-render as needed, based on the state.
  • The library takes care of DOM updates using the virtual DOM.

React concepts

State

  • A JavaScript object
  • A set of key/value pairs that represents the current state of the application.
var me = {

React concepts

Component

  • A self-contained and reusable bit of code
  • One “piece” of a UI

React concepts

Virtual DOM (vDOM)

  • A JavaScript object
  • Created by the library auto-magically
  • An in-memory representation of DOM nodes and elements
  • The library updates the virtual DOM - then updates the DOM only when necessary.

JSX

  • A short-hand language that lets you create JavaScript objects using an HTML-like syntax.
  • Generates JavaScript objects by matching HTML element names to components.

React concepts

JSX Example

  • To create a component called WelcomeText that generates the HTML: 
    <p>Welcome to the React class!</p>
    We could create a component like this: 
    var WelcomeText = function() {
	return (
		<p>Welcome to the React class!</p>
	);
};
    Which is the equivalent of: 
    var WelcomeText = React.createComponent({
	render: function render() {
		return React.createElement(
			"p",
			null,
			"Welcome to the React class!"
		);
	}
});

Making Components Dynamic

  • Components use ‘props’ or properties for dynamic bits of data and functionality.
  • Props are expressed with the same syntax as HTML attributes, i.e. name={someValue}
  • A prop can be set to any JS value
    • String, Array, Object, Function, etc.
    • Other React components

What is an object?

  • A way to describe concepts in an application or program.
    • Customer
    • Product
    • Order
  • A “thing” with properties and methods
    • Properties describe the object
    • Methods are things the object can do

How to make an object

With an object literal:

var contact0 = {
	firstName: 'Natalie',
	lastName: 'MacLees',
	organization: 'Girl Develop It'
};

Adding properties:

var contact1 = {};
contact1.firstName = 'Irina';
contact1.lastName = 'Muchnik';
contact1['organization'] = 'Girl Develop It';

Objects can be nested:

var contact2 = {
	firstName = 'Liz',
	lastName = 'Krane',
	organization: {
		name: 'Girl Develop It',
		website: 'http://girldevelopit.com'
	}
};

Accessing properties of objects

Two ways:


With dot notation:

contact0.firstName;
contact1.lastName;

With square bracket notation:

contact0['firstName'];
contact2['lastName'];

React stateful components

  • React components built with JSX just like before
  • Plus…
    • State
    • Mutation methods

A presentational component:

var Welcome = function(props) {
	return (
		<p>Welcome {props.firstName} {props.lastName}!</p>
	);
};

A stateful component:

var Welcome = React.createClass({
	getInitialState: function() {
		return {count: 10};
	},
	render: function() {
		return (
			<p>
				Welcome {this.props.firstName} {this.props.lastName}!
				Count: {this.state.count}
			</p>
		);
	}
})

The basics of state

To work with stateful components, you need to:

  • define getInitialState() to set the state (a simple object)
  • use this.setState(newState, callback) to update the state in mutations.
    • Learn more in the docs.
var MyComponent = React.createClass({
	getInitialState: function() {
		return {myKey: 1};
	},
	myFunc: function() {
		this.setState({myKey: ++this.state.myKey});
	},
	render: function() {
		return (
			<div>
				<span>Props: {this.props.myProp}</span>
				<span>Props: {this.state.myKey}</span>
				<button onClick={this.myFunc}>
					Trigger state!
				</button>
			</div>
		);
	}
});

ReactDOM.render(
	<MyComponent myProp={'some string'} />,
	document.getElementById('app')
);

Component Specifications

The component spec includes:

  • render() describes the component markup, and is the only required method
  • getInitialState() sets the starting state
  • getDefaultProps() defines the default prop values for all instances of a component
  • propTypes defines prop value constraints
  • Learn more in the docs.

Component Specifications

var MyComponent = React.createClass({

	// required
	render: function() { return <p>Hi!</p>; },

	// Invoked once before the component is mounted. The return value sets this.state.
	getInitialState: function() {
		return { count: 0 };
	},

	// Invoked once and cached when the class is created.
	getDefaultProps: function() {
		return { color: 'blue' };
	},

	// specifies constraints on props
	propTypes: {
		name: React.PropTypes.string.required,
		optionalNumber: React.PropTypes.number
	}
});

Component Lifecycle

Components have a lifecycle in three phases:

  1. Mounting: a new instance of a component will be added to a view
  2. Updating: rendering updates
  3. Unmounting: an instance of a component is removed from the view

Learn more about the lifecycle in the docs.

Component Lifecycle

var MyComponent = React.createClass({

	// required
	render: function() { return <p>Hi!</p>; },

	// Mounting: called once immediately before rendering
	componentWillMount: function() { ... },

	// Mounting: called once immediately after rendering
	componentDidMount: function() { ... },

	// Updating: called when new props are passed in
	// ** may happen on 2nd and subsequent renders
	componentWillReceiveProps: function(nextProps) {
		if (this.props.thing !== nextProps.thing ) {
			.. do something about it
		}
	},

	// Updating: Invoked before rendering when new props or state are being received.
	// return true to allow render, return false to prevent
	shouldComponentUpdate: function(nextProps, nextState) {
		return false;
	},

	// Updating: an update will occur
	componentWillUpdate: function(nextProps, nextState) { },

	// Updating: an update did occur
	componentDidUpdate: function(prevProps, prevState) { },

	// Unmounting:
	componentWillUnmount: function() {
	 // do clean up to avoid memory leaks!
	}
});

Forms!

Forms in React present some interesting challenges.

  • Unlike other elements, form controls can be mutated by users.
    • <input>
    • <select>
    • <textarea>


Try this example and see what happens!

Learn more about forms in the docs.

Form Inputs

Input controls in React provide a few props that are designed to keep the UI state information synchronized with the underlying DOM:

  • value: the current value of the input
  • defaultValue: used if the value is null or undefined
  • onChange: a general event handler, use instead of onInput

React Form Tips

  • Keep the form element's state in sync with React
    • Use the state to set the value of the form element
    • Use a method to update the state each time the value of the form element changes
  • Focus on one-way data flow
